Actúa como un Arquitecto de Software Senior y Desarrollador React. Vamos a mejorar el MVP de "BuroSE".

OBJETIVO:
Crear un dashboard de riesgo crediticio que fusione dos fuentes de datos:
1. Base de Datos Interna (Denuncias de usuarios).
2. Base de Datos Externa (BCRA - Banco Central de la República Argentina).

ESTRUCTURA TÉCNICA Y VISUAL:

1. TEMA ADAPTATIVO (System Preference):
   - Soporte nativo 'dark:' para Modo Claro/Oscuro.
   - Colores de Estado: Verde (Situación 1/Normal), Amarillo (Sit. 2), Rojo (Sit. 3, 4, 5 o Juicio).

2. LÓGICA DE DATOS (DATA MERGING):
   - Crea una función asíncrona `fetchRiskProfile(cuit)`.
   - Esta función debe simular una llamada a la API del BCRA (Central de Deudores) y traer:
     * Entidad (Ej: Banco Galicia, Tarjeta Naranja).
     * Periodo (Mes/Año).
     * Situación (Del 1 al 5).
     * Monto (en ARS).
   - Debe sumarizar la deuda total (Interna + BCRA) en un "Score de Riesgo" unificado.

3. INTERFAZ DE USUARIO (DASHBOARD):
   - Al buscar un CUIT, muestra un encabezado con el RIESGO TOTAL.
   - Al ingresar los números de cuit deberá ser sin guiones, tal cual lo solicita el bcra.
   - Debajo, divide la pantalla en dos paneles diferenciados:
     * PANEL IZQUIERDO: "Reportes del Gremio" (Denuncias de nuestros usuarios con fotos/evidencia).
     * PANEL DERECHO: "Situación Financiera Oficial (BCRA)" (Listado de bancos y estado de deuda).

4. CASOS DE USO A SIMULAR (MOCKS):
   - CUIT "20111111112": Usuario LIMPIO (Sin denuncias gremiales, BCRA Situación 1).
   - CUIT "30222222223": Usuario DEUDOR MIXTO.
     * Gremio: 1 Denuncia por factura impaga ($500k).
     * BCRA: "Banco Santander - Situación 3 (Con problemas) - $1.2M".
     * El sistema debe mostrar una ALERTA ROJA combinada.

Genera el código completo en React + Tailwind. Incluye la lógica para calcular la suma total de deuda de ambas fuentes.
